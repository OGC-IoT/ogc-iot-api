Sample Services

To demonstrate the OGC SensorThings API, we current have one live service for you to experiment with:


Sample Service Description
OGC SensorThings Sample Service V1.0 This sample service supports CRUD functions with simulated dataset. Please note that as this sample service serves the purpose of demonstration, the data in this service is reset to default every day.

Interactive SDKs

To help you to understand the requests and responses in the OGC SensorThings API, we currently have an interactive SDK for you to explore:


Interactive SDK Description
SensorThings API Interactive SDK This interactive SDK provides some examples of requests that can be sent to an OGC SensorThings service.

Applications

We currently have one very simple client for you to explore the sample service:


Application Description
Sample application This sample application provides a simple web user interface for users to browse the entities in an OGC SensorThings sample service. This sample application also showcases that charts and maps can be applied to represent observations and locations.

Hardware

The OGC SensorThings API is designed for resource constraint IoT devices. The API can be within the major IoT development platforms, such as Arduino, Netduino (.NET micro), Raspberry Pi (Linux), mbed, etc. Currently we have implemented the OGC SensorThings API using the .NET Micro Framework for the Netduino devices.
